    	<h3>Remarks</h3>
    	<p>The search performed by this method is not case-sensitive. The <code>s</code> parameter is a substring to compare against the text associated with the items in the combo box list. The search performs a partial match starting from the beginning of the text, and returning the first item in the list that matches the specified substring. You can then perform tasks, such as removing the item that contains the search text using the <a href="system.windows.forms.combobox.objectcollection.remove?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">Remove</a> method or changing the item's text. Once you have found the specified text, if you want to search for other instances of the text in the <a href="system.windows.forms.combobox?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">ComboBox</a>, you must use the version of the <a href="system.windows.forms.combobox.findstring?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">FindString</a> method that provides a parameter for specifying a starting index within the <a href="system.windows.forms.combobox?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">ComboBox</a>. If you want to perform a search for an exact word match instead of a partial match, use the <a href="system.windows.forms.combobox.findstringexact?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">FindStringExact</a> method.</p>

	<h4 class="propertyInfoTitle stack">Returns</h4>



	<div class="propertyInfo">
		<a href="system.int32?view=windowsdesktop-9.0" data-linktype="relative-path">Int32</a>
		
	</div>

	<p>The zero-based index of the first item found; returns -1 if no match is found, or 0 if the <code data-dev-comment-type="paramref">s</code> parameter specifies <a class="no-loc" href="system.string.empty?view=windowsdesktop-9.0#system-string-empty" data-linktype="relative-path">Empty</a>.</p>



	<h4 class="propertyInfoTitle stack">Exceptions</h4>
		<div class="propertyInfo">
		    <div class="subtle">
			    <a href="system.argumentoutofrangeexception?view=windowsdesktop-9.0" data-linktype="relative-path">ArgumentOutOfRangeException</a>
		    </div>
		    <p>The <code data-dev-comment-type="paramref">startIndex</code> is less than -1.</p>
<p>-or-</p>
<p>The <code data-dev-comment-type="paramref">startIndex</code> is greater than the last index in the collection.</p>

		</div>


    	<h3>Remarks</h3>
    	<p>The search performed by this method is not case-sensitive. The <code>s</code> parameter is a substring to compare against the text associated with the items in the combo box list. The search performs a partial match starting from the beginning of the text, returning the first item in the list that matches the specified substring. You can then perform tasks, such as removing the item that contains the search text using the <a href="system.windows.forms.combobox.objectcollection.remove?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">Remove</a> method or changing the item's text. This method is typically used after a call has been made using the version of this method that does not specify a starting index. Once an initial item has been found in the list, this method is typically used to find further instances of the search text by specifying the index position in the <code>startIndex</code> parameter of the item after the first found instance of the search text. If you want to perform a search for an exact word match instead of a partial match, use the <a href="system.windows.forms.combobox.findstringexact?view=windowsdesktop-9.0" class="no-loc" data-linktype="relative-path">FindStringExact</a> method.</p>
